# Implementation Plan: Admin List Hourly Available Slot Display

**Branch**: `012-set-list-default` | **Date**: 2026-04-16 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/012-set-list-default/spec.md`

## Summary

Change the available-slot display granularity in the admin list view from 30-minute blocks to 1-hour blocks. Reserved/booked slots continue to appear as a single merged row (feature 011 behaviour). The implementation is a single-constant change (`SLOT_STEP_MINUTES`: 30 → 60) plus partial-gap row emission for gaps shorter than 1 hour adjacent to non-hour-boundary bookings.

**Modified file**: `src/features/admin/calendar/deriveAdminListRows.ts` (no other files change)  
**No new npm packages, no Supabase migrations, no schema changes.**

## Implementation Detail

### `src/features/admin/calendar/deriveAdminListRows.ts` (MODIFIED)

**Change 1** — constant:
```typescript
// Before (011):
const SLOT_STEP_MINUTES = 30

// After (012):
const SLOT_STEP_MINUTES = 60
```

**Change 2** — partial gap before each booking (inside `for` loop, after the `while` block):
```typescript
// [NEW in 012] Emit partial gap row if remaining time before booking < 60 min
if (cursor < effectiveBStart) {
  const partialDuration = (effectiveBStart.getTime() - cursor.getTime()) / 60000
  rows.push({
    type: 'available',
    slotStart: cursor,
    slotEnd: effectiveBStart,
    durationMinutes: partialDuration,
    status: 'AVAILABLE',
    actionable: true,
  })
  cursor = effectiveBStart
}
```

**Change 3** — partial trailing gap (after the trailing `while` loop):
```typescript
// [NEW in 012] Emit partial trailing gap if any time remains before scheduleEnd
if (cursor < scheduleEnd) {
  const partialDuration = (scheduleEnd.getTime() - cursor.getTime()) / 60000
  rows.push({
    type: 'available',
    slotStart: cursor,
    slotEnd: scheduleEnd,
    durationMinutes: partialDuration,
    status: 'AVAILABLE',
    actionable: true,
  })
}
```
